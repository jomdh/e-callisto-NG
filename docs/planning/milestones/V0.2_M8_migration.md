# Milestone V0.2-M8 -- Legacy migration & interop

**Theme:** Turn "switch to NG" into a 5-minute import for an existing Callisto
station, and keep feeding the central archive byte-unchanged.

**Closes at:** v0.2.2 (v0.2 version close -> merge to main)

## Scope

- **Import wizard / Tools -> Import** (DESIGN 9a): parse a legacy install --
  `callisto.cfg` (instrument identity, coordinates, focus, gain, paths),
  `frqXXXXX.cfg` (-> frequency programs), `scheduler.cfg` (-> NG schedule rules),
  `CALxxxxx.prn` (-> calibration sets); **index existing FITS in place** (catalog
  + quicklooks), no re-upload. Preview-then-commit, non-destructive, idempotent.
- **Legacy output mode** (DESIGN 6a): byte-compatible filenames + header card
  set/order so data drops into the current archive unchanged; export
  `scheduler.cfg`. Per-instrument and per-upload-target mode selection (legacy /
  standard / custom), so one station can ship legacy to the archive and standard
  to a mirror.

## Sprints (planned)

- **S024** -- Import service + wizard branch (config/programs/schedule/cal +
  in-place FITS indexing).
- **S025** -- Legacy output writer + mode selection per instrument/target;

## Completion criteria

- [ ] A legacy install imports its identity, programs, schedule, calibration, and
  historical FITS without touching the original.
- [ ] Legacy-mode files match the archive's filename/header conventions.
- [ ] An upload target can request a specific output mode.
- [ ] Full gate green; tag v0.2.2; 0.2-dev merged to main (v0.2 released).
