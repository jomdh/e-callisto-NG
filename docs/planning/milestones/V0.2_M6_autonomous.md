# Milestone V0.2-M6 -- Autonomous operation

**Theme:** Make the station actually run a day unattended. v0.1 built every
piece but left the autonomy loop unwired: the scheduler computes windows but
nothing records on them, uploads are manual, calibration isn't applied.

**Closes at:** v0.2.0

## Scope

- **Scheduler drives recording.** A background scheduler service ticks each
  enabled schedule and starts/stops the recorder at the right UTC moments
  (DESIGN 1, 12). Needs a **continuous** recording mode (record until stopped /
  file-period rollover), not just bounded frames.
- **Auto-dispatch uploads.** Wire the dispatch modes (DESIGN 13): **immediate**
  (on each finished file) and **scheduled/overnight window**; manual stays.
- **Retention/pruning.** Delete local data after confirmed upload + by
  age/free-space; never delete un-uploaded data (DESIGN 13, 14a).
- **Calibration wiring.** Persist coefficient sets, assign per instrument; the
  recorder builds a calibrated `Recording` when a calibrated unit is chosen.
- **Light curves.** Emit the per-channel light-curve product (DESIGN 6).

## Sprints (planned)

- **S018** -- Scheduler service loop + continuous recording mode.
- **S019** -- Auto-dispatch uploads (immediate + window) + retention.
- **S020** -- Calibration persistence/assignment wiring + light-curve output.

## Completion criteria

- [ ] A configured station records on its sun schedule with no operator action.
- [ ] Finished files upload automatically per the target's dispatch mode.
- [ ] Local data is pruned after confirmed upload; un-uploaded data is safe.
- [ ] An instrument assigned a calibration writes SFU/Kelvin FITS.
- [ ] Light-curve files are produced; full gate green; tag v0.2.0.
