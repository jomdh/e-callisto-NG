# Milestone V0.2-M7 -- Security & deployment hardening

**Theme:** Make the station safe to expose and simple to ship. v0.1 is LAN-only
with plaintext credentials and no CSP -- not field-safe.

**Closes at:** v0.2.1
**Release gate:** B2 (credential encryption) and CSP are **blocking** -- v0.2
does not ship until both hold (operator directive).

## Scope

- **B2: encrypt upload credentials at rest** (DESIGN 10). Machine-bound key;
  encrypt/decrypt the `UploadTarget` secret fields; never return plaintext via
  the API; write-only in the UI.
- **CSP middleware + nonce** (the portal's own rule). `script-src 'nonce-...'`;
  move the theme-bootstrap inline script to a nonce; audit for inline handlers.
- **Remote-access modes** (DESIGN 10): LAN/VPN-only (default), public-HTTPS +
  **dynamic-DNS** updater, and outbound **reverse-tunnel/relay**; access settings
  UI; reachability test. Caddy config generation.
- **Real clock-sync probe** (DESIGN 12a): query `timedatectl`/chrony for true
  NTP sync; feed health (replace the tri-state placeholder) and the recording
  gate.
- **Full `.deb`**: `debian/` packaging (control, rules, postinst creating the
  `callisto` user + dirs, installs the unit). Build with `dpkg-buildpackage`.

## Sprints (planned)

- **S021** -- B2 credential encryption + CSP middleware/nonce (the release gate).
- **S022** -- Remote-access modes + dynamic-DNS + reverse-tunnel + access UI.
- **S023** -- Clock-sync probe into health/gate + full `.deb` packaging.

## Completion criteria

- [ ] Upload credentials are encrypted at rest; never returned in plaintext.
- [ ] CSP enforced; no inline handlers; theme bootstrap nonced.
- [ ] Operator can choose LAN / public-HTTPS+DDNS / reverse-tunnel; reachability
  verified.
- [ ] Health reflects real NTP sync; recording gates on drift.
- [ ] `.deb` builds and installs cleanly; full gate green; tag v0.2.1.
