# Milestone M15 — Station Settings (depth)

**Closes:** v0.5.0  **Branch:** `0.5-dev`  **Theme:** complete the §8.4 Station
Settings pillar -- everything about the host and the suite, not the science.
(Design proposal "M12"; renumbered after v0.4 parity.)

## Deliverables

| # | Area (DESIGN §8.4) | Deliverable |
| -- | -- | -- |
| D1 | **Users & Access** | user CRUD UI (create/disable/role), password policy, session list, **audit log** of security-sensitive actions |
| D2 | **Time** | NTP/chrony status + drift detail page; enforcement policy (uses the M14-watchdog `require_clock_sync`) |
| D3 | **Storage** | disk usage breakdown, retention-rule editor, backup-location setting |
| D4 | **Software** | version display, config **backup/restore** (export/import the station's DB-backed config), update-channel scaffolding (the runner lands in M17) |
| D5 | **System** + **data browser depth** | journald **log viewer**, receiver reconnect, reboot/shutdown (guarded); plus the richer data browser folded from the old proposal -- calendar/heatmap, in-browser FITS viewer, bulk download/delete/re-queue (§8.1) |

## Notes

- Audit log is a new append-only table; writing it is a security-sensitive
  change -> ADR.
- Reboot/shutdown are host-level: guard behind admin + an explicit confirm, and
  shell out via a documented, least-privilege hook (no arbitrary command exec).

## Acceptance

- Admin manages users + sees an audit trail; viewers/operators cannot.
- Time/Storage/Software/System pages render real host data.
- Config backup round-trips (export then import reproduces settings).
- Data browser supports calendar view + in-browser FITS + bulk ops.
- Quality gate green; SNR clean; tag v0.5.0.

## Out of scope

The update *runner* + rollback + SD image (M17); resilience model (M16).
