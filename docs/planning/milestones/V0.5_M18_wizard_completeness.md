# Milestone M18 — Wizard completeness

**Closes:** v0.5.3 / v0.5 version close  **Branch:** `0.5-dev`  **Theme:** the §9
full first-run experience -- a guided, resumable flow, then merge to `main`.
(Design proposal "M16". F9.)

## Deliverables

| # | DESIGN ref | Deliverable |
| -- | -- | -- |
| D1 | §9 multi-step resumable | full **multi-step** wizard (admin -> station identity -> coordinates -> instrument -> program -> schedule -> access -> review), **resumable** via `settings.wizard_state` across refresh/reboot |
| D2 | §9 coordinates | **map picker** for lat/long/alt (offline-friendly), cloneable across instruments |
| D3 | §9a clone/import branch | wizard **branch** to provision from an existing station (clone config) or **import** a legacy station (reuses the M8 import) |
| D4 | §8.2 "wizard = guided Configurator" | the wizard steps **are** the permanent per-instrument editors (one set of screens, two entry points) |

## Notes

- Builds on the existing minimal wizard; the steps reuse the M15/Configurator
  forms so there is no duplicate UI.
- Resumability: persist step state server-side; the app redirects to the wizard
  until complete (already the case) and resumes mid-flow.

## Acceptance

- A fresh station completes setup through every step; a refresh mid-flow resumes.
- The map picker sets coordinates; clone/import branch provisions a second station.
- Editing an instrument later uses the same screens.
- Quality gate green; SNR clean; tag v0.5.3; merge `0.5-dev` -> `main`.
