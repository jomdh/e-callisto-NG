# Milestone M22 — doncel.dev integration

**Closes:** v0.7.2  **Branch:** `0.7-dev`  **Theme:** the design's stated
end-goal (§1): NG integrates with the doncel analysis tier **via API + data**
(the shared design system already landed in M19).

## Deliverables
| # | Deliverable |
| -- | -- |
| D1 | **Push contract**: NG publishes its station catalog (recordings index + metadata) + health to a configured doncel endpoint, on a schedule, behind an `IntegrationTarget` (URL + token) |
| D2 | **Pull/registration**: doncel registers an NG station as a data source; NG exposes a stable read API (catalog/health) doncel can poll (token-gated, like fleet-health) |
| D3 | A `services/integration` module behind a contract + a fake doncel endpoint for tests; retry/offline-queue like the uploader |
| D4 | Settings UI: configure the doncel endpoint, token, what to share; test connection |
