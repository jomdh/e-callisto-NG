# Milestone M12 — Bench & diagnostic tools

**Closes:** v0.4.1  **Branch:** `0.4-dev`  **Theme:** the legacy bench/commissioning
apps (`simple`, `NoiseFigurePlotter`), web-native. Maps `WINDOWS_FUNCTIONALITY.md`
§2 (simple, NF) and the design §8.3 Tools pillar.

## Parity gaps -> deliverables

| # | Legacy tool (evidence) | Deliverable |
| -- | -- | -- |
| D1 | **simple**: tune one frequency `F0`, set gain `O`, poll detector voltage `A0`, show on a bar with an audible trigger threshold (`simple/`) | Bench/Detector tool: driver `tune`/`set_gain`/`read_detector` (A0); live voltage readout + threshold indicator in Tools |
| D2 | **NF Y-factor**: hot/cold/warm noise-figure, detector slope (mV/dB) (`NoiseFigurePlotter/mainunit.cpp`) | pure Y-factor noise-figure + slope math; driver capture; Tools page with the computed NF |
| D3 | **NF sweeps**: bandpass, AGC linearity, time-domain digitizer/scope; relay/focus switching `fs` (`NoiseFigurePlotter/`) | bandpass + AGC-linearity sweeps; focus/relay switch; scope capture; charts |
| D4 | Plotting/export (DISLIN -> web) | Tools UI: charts (canvas island) + CSV/PNG export of each measurement |

## Approach

- Driver gains bench primitives behind the contract (extend `InstrumentDriver`
  or a `BenchCapable` optional protocol -- ADR if it touches the contract).
- All measurement math is **pure + tested** (Y-factor, slope, NF); the hardware
  capture is a thin wrapper, faked by the serial simulator.
- New Tools pages reuse the JS-island + API pattern from S029.

## Acceptance

- Detector-voltage readout works against the simulator.
- Y-factor NF + detector slope computed correctly (pure-math tests).
- Bandpass/AGC sweeps + focus switch run; results chart + export.
- Quality gate green; SNR clean; tag v0.4.1.

## Out of scope

Spectrum viewer + public PNG (M13); distribution (M14).
